dhtmlXFolders Built-in Item Types

Type Name XSL file Xml file format sample Variables used in xsl template
ficon ficon.xsl <data> <userdata name="[unique name]">value</userdata> <item id="[unique id]" name="[file name with extension]" /> <item .../> </data> icons_src_dir - path to directory with icons reffering to the page
ftable ftable.xsl <data> <userdata name="[unique name]">value</userdata> <item id="[unique id]" name="[file name with extension]" > <filesize>[size in bytes]</filesize> <modifdate>[formatted date]</modifdate> </item> <item> ... </item> </data> icons_src_dir - path to directory with icons reffering to the page
fthumbs fthumbs.xsl <data> <userdata name="[unique name]">value</userdata> <item id="[unique id]" name="[file name with extension]" /> <item> ... </item> </data> icons_src_dir - path to directory with icons reffering to the page thumbs_creator_url - url of thumbnails creator file photos_rel_dir - directory with photos
ftiles ftiles.xsl <data> <userdata name="[unique name]">value</userdata> <item id="[unique id]" name="[file name with extension]" > <filesize>[size in bytes]</filesize> </item> <item> ... </item> </data> icons_src_dir - path to directory with icons reffering to the page

* - simpler types, like ficon can contain additional tags and attributes which can be used then for some additional purposes out of dhtmlxFolders object -like output of additonal information for selected item, or inside dhtmlxFOlders object - like sorting by some invisible fields - modififcation date, size etc. In general, 4 types: ficon, ftable, fthumbs, ftiles can use same XML file as datasource. This is demonstrated in sample application "Image browser" (find it among samples)
